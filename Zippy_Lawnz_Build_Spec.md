## 1. Project Overview

### 1.1 Product

A mobile-first web app at `zippylawnz.com` (or chosen domain). Anonymous-by-default flow with optional account creation at lead-capture step. Users complete a yard assessment in <10 minutes and receive a ranked list of compatible robotic mowers plus a path to purchase/install through ZippyLawnz.

### 1.2 Primary Personas

- **Homeowner (P1)** — researching first robotic mower purchase. Mobile user, low technical skill, mid-to-high income, owns home with ≥0.1 acre lawn.
- **ZippyLawnz Sales Rep (P2)** — uses the same app on tablet during in-home consultations. Needs faster flow, ability to override CV detections, save assessments to a customer record.

### 1.3 Out of Scope (V1)

- Native mobile apps (web-first; PWA-ready)
- Real-time mower control / post-purchase telemetry
- Marketplace transactions (payment processing for mowers)
- Non-U.S. markets, non-English UI
- Automated landscape design (Phase 3)

### 1.4 Success Metrics

- Assessment completion rate ≥ 60% of starts
- Lead-capture conversion ≥ 25% of completed assessments
- ZippyLawnz qualified-lead-to-sale conversion ≥ 15%
- Median assessment time ≤ 8 minutes
- Slope measurement within ±3° of professional inclinometer (validated against 20 test yards)

## 2. Tech Stack & Conventions

### 2.1 Stack

| Layer | Technology | Notes |
|---|---|---|
| Frontend | Next.js 15 (App Router), React 19, TypeScript 5+, Tailwind CSS 4 | Mobile-first; PWA manifest |
| UI components | shadcn/ui | Don't build primitives from scratch |
| Maps (interactive) | Mapbox GL JS v3+ | Drawing layer, polygon editing |
| Maps (imagery) | Google Maps Static API | Server-side proxy only |
| Backend | Python 3.12+, FastAPI, Pydantic v2 | Async-first |
| ORM | SQLAlchemy 2.0 + Alembic | Type-stub generation enabled |
| Database | PostgreSQL 16 + PostGIS 3.4 | Yards stored as `GEOGRAPHY(POLYGON, 4326)` |
| Cache/Queue | Redis 7 | Sessions + background job queue |
| Storage | Cloudflare R2 (S3-compatible) | User photos, captured map snapshots |
| ML inference | Replicate (SAM 2 + custom grass classifier) | Pay-per-call; abstract behind service interface |
| Auth | Clerk | Email + Google OAuth at MVP |
| Email | Resend | Transactional only at MVP |
| Hosting | Railway | Frontend, backend, Postgres, Redis |
| Monitoring | Sentry (errors), PostHog (product analytics) | Both have generous free tiers |

### 2.2 Code Conventions

- **TypeScript:** strict mode on. No `any`. Zod for runtime validation at API boundaries.
- **Python:** type hints everywhere. `ruff` + `mypy --strict` in CI.
- **Naming:** `snake_case` for Python, `camelCase` for TS, `PascalCase` for components, `kebab-case` for files and routes.
- **Commits:** Conventional Commits format (`feat:`, `fix:`, `chore:`, etc.).
- **Branches:** trunk-based. `main` is always deployable. Feature work in short-lived `feat/...` branches.
- **API contract:** OpenAPI spec auto-generated by FastAPI is the source of truth. Frontend types generated from it.
- **Secrets:** never committed. Railway env vars + `.env.local` for dev. Use `.env.example` to document required vars.

### 2.3 Environments

- `dev` — local, Docker Compose for Postgres + Redis
- `staging` — Railway, mirrors prod, separate DB
- `prod` — Railway, real Google Maps key, real Replicate key

## 5. API Surface

All endpoints prefixed `/api/v1`. JSON only. Bearer token auth via Clerk JWT for authenticated routes; anonymous routes use a signed `session_id` cookie.

### 5.1 Public/Anonymous

```
POST   /assessments                       # create draft, returns id + session token
GET    /assessments/{id}                  # fetch (must match session or auth)
PATCH  /assessments/{id}                  # update fields (incremental save)
POST   /assessments/{id}/geocode          # body: {address}; returns {lat, lng, suggested_zoom}
POST   /assessments/{id}/lawn-segment     # body: {map_image_url, click_points}; calls SAM 2 via Replicate
POST   /assessments/{id}/slope-sample     # body: {lat, lng, angle, accuracy}
POST   /assessments/{id}/grass-photo      # multipart upload; returns top 3 species guesses
POST   /assessments/{id}/complete         # finalizes + computes recommendations
GET    /assessments/{id}/recommendations  # ranked mower list with reasons

POST   /leads                             # body: {assessment_id, email, ...}
GET    /mowers                            # public mower catalog with filters
GET    /mowers/{slug}                     # mower detail
```

### 5.2 Authenticated (Homeowner)

```
GET    /me/assessments                    # list user's saved assessments
DELETE /me/assessments/{id}
```

### 5.3 Authenticated (Sales Rep)

```
GET    /sales/leads?status=new            # ZippyLawnz CRM-lite
PATCH  /sales/leads/{id}                  # update status, notes
GET    /sales/assessments/{id}            # any assessment by ID for sales context
```

### 5.4 Admin

```

```

### 5.5 Internal / Webhooks


## 6. Phase 1 — MVP (Plan & Choose)

**Goal:** A homeowner with no prior knowledge can go from landing page to receiving 3 ranked mower recommendations in under 10 minutes, and ZippyLawnz receives a qualified lead.

**Target ship:** 8–12 weeks.

### 6.1 User Stories

| ID | As a... | I want to... | So that... |
|---|---|---|---|
| US-1 | homeowner | enter my address and see my house from above | I can confirm we're looking at the right property |
| US-2 | homeowner | mark the boundary of my lawn on a satellite image | I get an accurate area measurement |
| US-3 | homeowner | walk my hill with my phone to measure slope | I know if my yard is too steep for a robotic mower |
| US-4 | homeowner | snap a photo of my grass and learn what type I have | I get the right cutting recommendations |
| US-5 | homeowner | tag my gates and obstacles | the recommended mower will actually fit through and around them |
| US-6 | homeowner | see which mowers will work for my yard, ranked, with reasons | I can confidently pick one |
| US-7 | homeowner | save my report and request a free consultation | ZippyLawnz can help me complete the purchase and install |
| US-8 | sales rep | open any saved assessment by ID and review it | I can have an informed sales conversation |
| US-9 | admin | add, edit, and remove mowers in the catalog | the recommendations stay current |

### 6.2 Feature Detail

#### 6.2.1 Landing & Marketing Pages

- Hero with single CTA: "Start your assessment"
- Three-step explainer: Map your yard → Measure your slope → Match your mower
- FAQ section
- Trust strip: "Powered by ZippyLawnz"
- Footer with links to ZippyLawnz parent site

**Acceptance:** Lighthouse mobile score ≥ 90 across all categories.

#### 6.2.2 Address Entry

- Single input with Google Places autocomplete (US addresses only)
- Validates address resolves to a residential lat/lng
- On submit: creates `Assessment` draft, geocodes, advances to map step
- Allows manual lat/lng override for sales rep mode

**Acceptance:** 95% of valid U.S. residential addresses resolve correctly. Edge cases (PO boxes, unbuilt lots) show a clear error message.

#### 6.2.3 Satellite View Positioning

- Mapbox GL canvas centered on geocoded lat/lng, zoom 19, satellite-imagery style backed by Google tiles via server-side proxy
- User can pan, zoom, rotate the view
- "This is my house" confirmation button to lock the view
- Captures `{center, zoom, bearing}` to the assessment record (NOT a screenshot — keep geographic precision)

**Acceptance:** Map renders within 2s on a mid-tier Android phone over 4G. Pan/zoom is 60fps.

#### 6.2.4 CV Lawn Boundary Detection

- After view is locked, user taps anywhere on grass → backend calls SAM 2 via Replicate
- Backend posts a rendered map tile + click point to Replicate; receives a polygon mask
- Polygon converted from pixel coordinates to lat/lng using the locked map view's projection
- Stored as PostGIS `GEOGRAPHY(POLYGON, 4326)`
- Frontend displays the polygon as an editable overlay; user can:
  - Add/remove vertices
  - Click another grass area to add a second polygon (multi-zone)
  - Tap inside an existing polygon to "subtract" (no-grass island)
- Live area readout in sq ft updates on every edit

**Acceptance:**
- SAM 2 call returns within 8s p95
- Auto-detected polygon requires ≤ 5 user adjustments on 80% of test yards
- Final area is within ±10% of ground truth on a benchmark of 20 test yards

**Failure handling:** if Replicate is down or returns an error, fall back to manual polygon-drawing tool. Never block the user.

#### 6.2.5 Slope Measurement

- New screen with instructions and an animated diagram: "Hold your phone flat against the slope, walk to the steepest part, tap to record"
- Uses `DeviceOrientationEvent` (iOS requires permission prompt — handle that explicitly)
- "Record" button captures the current pitch angle plus geolocation
- User records 3–10 sample points; UI encourages sampling steep areas
- Computes max and average slope %; displays a visual color-coded slope indicator
- Allows skipping with a "my yard is flat" option that records 0% with a flag

**Acceptance:**
- Sensor reading accuracy ±2° vs. a physical inclinometer in calibration tests
- iOS 13+ permission prompt handled with clear UX
- Graceful fallback for desktop browsers (manual entry)

#### 6.2.6 Grass Type Identification

- "Take a photo of your grass" screen with framing guide overlay (close-up, 4–6 inches from blades)
- Photo uploaded to R2; metadata stored on assessment
- Backend calls grass classification model on Replicate
- Returns top 3 species with confidence scores; displayed as cards user can confirm or override
- Manually-selectable list of common U.S. species: Bermuda, Kentucky Bluegrass, Tall Fescue, Fine Fescue, Perennial Ryegrass, St. Augustine, Zoysia, Centipede, Bahia, Buffalo

**Acceptance:**
- Top-3 accuracy ≥ 75% on a benchmark of 100 labeled grass photos
- "I don't know" / manual selection option always available
- Photo upload + classification completes in ≤ 6s p95

**MVP shortcut:** if the custom grass classifier isn't ready, ship with a manual dropdown only and a "photo coming soon" message. Don't block launch on this.

#### 6.2.7 Gates & Obstacles

- Returns user to the map view with their lawn polygon overlaid
- Toolbar with: "Add gate", "Add obstacle", "Add no-go zone"
- Gate: drop pin + enter width in inches (allow tape-measure or estimate)
- Obstacle: drop pin + select type (tree, flower bed, pool, sprinkler head, slope-too-steep, other)
- No-go zone: draw polygon
- All saved to assessment record

**Acceptance:** clear undo/redo. Pins draggable. Mobile-friendly tap targets ≥ 44pt.

#### 6.2.8 Mower Recommendation Engine

A pure function in the backend: `compute_recommendations(assessment) -> List[Recommendation]`.

**Inputs from assessment:**
- `lawn_area_sqft`
- `max_slope_pct`
- `min_gate_width_inches` (computed from gates)
- `grass_type` (for cutting-height match)
- `multi_zone_count`

**For each active `Mower`:**

```python
fit_score = 100
reasons = []

# Hard gates (red)
if assessment.lawn_area_sqft > mower.max_area_sqft * 1.0:
    fit_score = 0
    reasons.append({type: 'area_too_large', severity: 'red', ...})

if assessment.max_slope_pct > mower.max_slope_pct:
    fit_score = 0
    reasons.append({type: 'slope_too_steep', severity: 'red', ...})

if assessment.min_gate_width_inches < mower.min_passage_inches:
    fit_score = 0
    reasons.append({type: 'gate_too_narrow', severity: 'red', ...})

# Soft factors (yellow)
if assessment.lawn_area_sqft > mower.max_area_sqft * 0.85:
    fit_score -= 15
    reasons.append({type: 'area_near_limit', severity: 'yellow', ...})

if assessment.max_slope_pct > mower.max_slope_pct * 0.85:
    fit_score -= 15
    reasons.append({type: 'slope_near_limit', severity: 'yellow', ...})

# Bonuses (green)
if mower.drive_type == 'awd' and assessment.max_slope_pct > 25:
    fit_score += 5
    reasons.append({type: 'awd_advantage', severity: 'green', ...})

# Status from final score
if fit_score == 0: status = 'red'
elif fit_score < 70: status = 'yellow'
else: status = 'green'
```

Sort by `fit_score desc`, then `price_usd asc`. Persist top 10 as `Recommendation` rows.

**Acceptance:** unit tests covering all branches; recommendations regenerate if assessment is edited.

#### 6.2.9 Results Page

- Top section: yard summary (area, max slope, grass, gates, obstacle count) with a small map preview
- Three tabs: ✅ Will Work | ⚠️ Borderline | ❌ Not Recommended
- Each mower card: photo, brand+model, price, fit score, top 2 reasons (e.g., "Handles your 28% slope easily")
- Click into mower detail page (full specs, manufacturer link, install considerations)
- Sticky CTA at bottom: "Get a free consultation from ZippyLawnz" → opens lead-capture modal

**Acceptance:** results render within 1s of `/recommendations` response. CTA conversion tracked in PostHog.

#### 6.2.10 Lead Capture

- Modal: "Get expert help completing your purchase and install"
- Fields: email (required), phone (optional), preferred contact, notes
- On submit: creates `Lead` row, sends transactional email to user with PDF report attached, sends notification email to ZippyLawnz sales inbox
- Success state offers: "Save this assessment to your account" (Clerk signup)

**Acceptance:** email deliverability validated against major providers (Gmail, Outlook, Yahoo). Lead row links back to assessment for sales rep context.

#### 6.2.11 Sales Rep Mode (Minimal)

- Sales reps log in via Clerk; role assigned via Clerk publicMetadata
- Rep can run an assessment without lead capture (skip the modal)
- Rep dashboard: paginated list of new leads, click to view full assessment
- Status update on lead (`new → contacted → qualified → sold | lost`) with notes

**Acceptance:** access control verified — homeowners cannot reach `/sales/*` routes.

#### 6.2.12 Admin: Mower Catalog

- Authenticated admin route at `/admin/mowers`
- List, create, edit, soft-delete mowers
- CSV bulk import endpoint with dry-run mode
- Image upload to R2

**Acceptance:** seed data includes 15 mowers spanning the major brands at MVP launch.

### 6.3 MVP Initial Mower Catalog (seed data)

To populate at launch — exact specs to be verified against current manufacturer spec sheets at build time. The catalog is data, not code; it must be easy to update.

Brands to cover: Husqvarna, Segway Navimow, Mammotion, ECOVACS, eufy, Greenworks, Yarbo, RoboUP, Sunseeker, ANTHBOT, MOVA, Lymow, YardCare, Worx, Stihl. Pick the top 1–2 models per brand by current reviews/sales.

### 6.4 Out of Scope for MVP

Explicitly deferred to Phase 2+:

- Multi-zone transport planning
- Training/install video library
- ROI calculator

## 7. Phase 2 — Install & Own

**Goal:** Extend the journey beyond the buying decision. Help users install successfully and get ongoing value from their mower.

**Target ship:** 3–6 months post-MVP.

### 7.1 Features

#### 7.1.1 Battery & Runtime Calculator
- Input: chosen mower + assessment data
- Output: estimated runtime per session, sessions per week to maintain lawn, expected battery life over years
- Logic: `runtime = (lawn_area_sqft / mower.cutting_rate_sqft_per_hour) * slope_penalty_factor`

#### 7.1.2 Multi-Zone Transport Planning
- For yards with disconnected mowable areas, identify transport paths
- Flag whether the mower supports auto-zone-switching or requires manual moves
- Show on map with directional arrows

#### 7.1.3 Training & Install Video Library
- Per-mower video collection: unboxing, base station placement, RTK setup, gate cuts, wire boundary install (for wired models), zone setup, troubleshooting
- Searchable, taggable
- Hosted via Mux or YouTube unlisted
- Track completion per user account

#### 7.1.4 Gate-Cutting & Base-Station Placement Guides
- Interactive guide using the user's own yard map
- Recommended base station location based on RTK signal coverage rules
- Gate-cut height/width specifications per mower

#### 7.1.5 ROI Calculator
- Inputs: current lawn-care monthly spend (mowing service, gas, time)
- Outputs: payback period, 5-year savings, hours/year reclaimed
- Includes battery replacement cost in TCO

#### 7.1.6 Save & Share Reports
- PDF generation server-side via Playwright
- Branded with ZippyLawnz / ZippyLawnz
- Includes yard map, slope chart, recommendations, ROI summary
- Shareable link option (read-only public URL)

#### 7.1.7 Affiliate Integration
- Add `affiliate_url` to mower records
- Track click-throughs in analytics
- Disclosure banner per FTC requirements
- A/B test affiliate vs. ZippyLawnz CTA placement

#### 7.1.8 User Accounts & Saved Assessments
- Full Clerk integration (already at MVP)
- "My Yards" dashboard for users with multiple properties (vacation home, parents' yard, etc.)
- Re-run assessment after seasonal changes

## 9. Non-Functional Requirements

### 9.1 Performance
- Frontend Lighthouse mobile score ≥ 90 across all categories on landing pages
- Time-to-interactive on assessment flow ≤ 3s on mid-tier Android over 4G
- API p95 latency < 400ms (excluding ML inference calls)
- ML inference fronted by async job pattern for any call > 2s

### 9.2 Reliability
- 99.5% uptime SLO (MVP); 99.9% by Phase 2
- All user input incrementally saved server-side; no data loss on browser crash
- Graceful degradation when Replicate, Mapbox, or Google Maps is unavailable

### 9.3 Security
- TLS everywhere; HSTS enabled
- All Google Maps and Replicate API calls go through backend; keys never in client
- Rate limiting per IP and per session_id
- Pen-test before public launch
- OWASP Top 10 compliance review

### 9.4 Privacy & Compliance
- U.S.-only at launch but build with GDPR-style data export/delete in mind
- Privacy policy disclosing CV processing, satellite imagery use, analytics
- Cookie consent banner
- No selling of user data; aggregate-only insights products
- TCPA compliance for any phone/SMS contact (Phase 2+)

### 9.5 Accessibility
- WCAG 2.1 AA target on assessment flow and results
- Keyboard navigation for all interactions
- Screen-reader labels on map controls
- Color is never the sole indicator (red/yellow/green also have icons + text)

### 9.6 Browser Support
- Latest 2 versions of Safari (iOS + macOS), Chrome (Android + desktop), Edge, Firefox
- Mobile Safari 14+ for DeviceOrientation
- Graceful messaging on unsupported browsers

### 9.7 Observability
- Sentry on both apps for errors
- PostHog for product analytics + funnel tracking
- Structured JSON logging on backend
- Custom dashboards: assessment funnel, ML inference cost, recommendation accuracy
